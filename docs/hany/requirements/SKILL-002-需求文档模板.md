# Skill 002：需求文档规范化

## 概述

本文档定义 hany-skills 技能集中的需求文档规范，确保所有技能执行时产出统一格式的需求文档。

## 目标

规范化 skill 002 的需求文档结构，要求：
1. 所有技能执行后必须生成需求文档
2. 需求文档格式统一、结构完整
3. 文档存储至 `docs/hany/requirements/` 目录

## 输入

| 来源 | 描述 |
|------|------|
| 用户原始描述 | 用户通过 `/hany:auto` 或 `/hany:require` 触发的任务描述 |
| 项目上下文 | 项目结构、技术栈、已有代码 |
| 对话澄清 | 多轮提问后的确认信息 |

## 需求文档标准格式

```markdown
---
status: in_progress
author: [作者]
version: "[版本]"
created: [日期]
---

# [主题名称]

## 目标
[一句话描述最终想要什么]

## 详细需求
### 需求项 1
[描述]

### 需求项 2
[描述]

## 输入/输出
| 类型 | 描述 |
|------|------|
| 输入 | [描述] |
| 输出 | [描述] |

## 技术约束
- [约束1]
- [约束2]

## 业务规则
- [规则1]
- [规则2]

## 异常处理
| 场景 | 处理方式 |
|------|---------|
| [场景] | [处理] |

## 不在范围内
- [排除的功能]

## 待澄清项
- [NEEDS CLARIFICATION: 具体问题]

## 歧义评分
| 维度 | 评分 |
|------|------|
| 目标清晰度 | X% |
| 约束清晰度 | X% |
| 验收标准 | X% |
| 综合 | X% |

## 选中方案
[方案名称 + 理由]
```

## 存储路径

- 路径：`docs/hany/requirements/YYYY-MM-DD-<主题>.md`
- 命名规范：日期-简短主题描述
- 示例：`docs/hany/requirements/2026-04-12-skill-002-需求文档规范.md`

## 质量标准

| 检查项 | 要求 |
|--------|------|
| status 标记 | 完成后改为 `completed` |
| 无占位符 | TBD/TODO/待定 必须消除 |
| 歧义度 | 综合评分 < 30% |
| 可验证 | 每项需求有明确的验收标准 |

## 触发时机

根据 `hany-require/SKILL.md` Step 9：
- 需求澄清完成后写入 `docs/hany/requirements/`
- 使用 `AskUserQuestion` 确认写入

## 归档

完成后将 `status: in_progress` 更新为 `status: completed`。
