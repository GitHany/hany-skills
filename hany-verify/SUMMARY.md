# hany-verify 摘要

**用途**：检查测试
**流程**：检查前置 → Agent独立验证 → 生成质量清单 → 3D验证 → 多视角并行审查（架构+安全+代码质量） → 功能验证（边找边报） → 回归验证 → 边界检查 → 发现问题立即修复（根因分析+错误→规则抽象） → 验证/修复循环（含质疑审查） → 优化反馈 → 输出报告 → 归档 → 最终确认
**输出**：`docs/hany/archive/YYYY-MM-DD-<主题>-verify.md`（验证报告）+ 修复的代码 + `docs/hany/rules.md`（错误规则）
**关键机制**：不信任Agent报告（必须独立验证）、边找边报（发现问题立即展示）、系统调试协议（4阶段）、验证/修复循环（standard最多3轮）、质疑审查（"你确定这真的对吗？"）、错误→规则抽象
**前置条件**：docs/hany/plans/ 和 docs/hany/requirements/ 下存在确认的文档
**质量模式**：strict(无上限)/standard(3轮)/minimal(1轮)
