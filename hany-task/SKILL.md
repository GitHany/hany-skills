# hany-task — AI Agent 任务分配检查助手

**触发方式：`/hany:task`**

> **编码约束**：读取 `hany-common/rules/rules.md`，11 条规则全程生效。

> **核心原则：量化评估，发现瓶颈，提供优化。通过标准化流程检查任务分配的合理性。**

## 概览

本 skill 用于检查 AI Agent 任务分配的合理性。它通过三个维度进行评估：
1. **工作负载均衡**：检查各 Agent 的任务量是否平衡。
2. **优先级合理性**：检查关键路径任务是否被正确识别和优先处理。
3. **依赖关系处理**：检查任务间的依赖是否导致不必要的阻塞或空闲。

## 工作流程

### Step 1：收集信息

获取以下信息：
- **Agent 列表**：参与任务的 Agent 名称/ID。
- **任务列表**：所有待分配的任务。
- **任务属性**：
  - 预估时间/复杂度
  - 优先级（如有）
  - 依赖关系（Task B 依赖 Task A）
- **当前分配方案**（可选）：如果已有方案，进行检查；否则进行预分配建议。

### Step 2：工作负载分析

计算每个 Agent 的总负载（任务时间之和）。

**检查指标：**
- **负载极差**：最大负载 - 最小负载。
- **负载方差**：衡量负载分布的均匀程度。
- **建议**：极差不应超过平均负载的 30%。

### Step 3：依赖关系与时序分析

构建依赖图，识别关键路径。

**关键步骤：**
1. **识别阻塞点**：找出被多个后续任务依赖的前置任务。
2. **计算最早开始时间**：考虑依赖关系后，每个任务的最早开始时间。
3. **识别空闲时间**：Agent 在等待依赖完成时的空闲时间。

**红灯信号：**
- 某个 Agent 长时间空闲，而其他 Agent 负载过重。
- 关键路径上的任务被分配给处理速度较慢或负载较重的 Agent。

### Step 4：优先级评估

检查优先级设置是否符合业务逻辑和依赖关系。

**原则：**
- **依赖优先**：被依赖的任务应具有较高优先级。
- **关键路径优先**：位于关键路径上的任务应优先处理。
- **高价值优先**：业务价值高的任务应优先。

### Step 5：生成优化建议

基于以上分析，生成具体的优化建议。

**建议类型：**
1. **重新分配**：建议将特定任务移动到特定 Agent。
2. **拆分任务**：建议将大任务拆分以提高并发度。
3. **调整优先级**：建议调整任务优先级以优化流程。

## 常见陷阱

1. **只看总时间，不看并发**：两个 Agent 总时间相同，但一个串行阻塞，一个完全并行，效率天差地别。
2. **忽视依赖深度**：深层依赖链会导致下游 Agent 长时间等待。
3. **平均主义**：强行平均分配任务数，而不考虑任务复杂度和依赖关系。
